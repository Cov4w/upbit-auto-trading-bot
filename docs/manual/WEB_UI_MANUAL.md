# 🎛️ Web UI 사용 설명서 (Web UI Manual)

Self-Evolving Trading System의 웹 인터페이스 사용법을 안내합니다.

---

## 1. 제어 패널 (Control Center)

봇의 핵심 기능을 제어하는 영역입니다.

*   **▶️ Start Bot:** 봇을 시작합니다. (백그라운드에서 매매 로직이 돕니다)
*   **⏸️ Stop Bot:** 봇을 중지합니다. (보유 포지션은 그대로 유지됩니다)
*   **🔄 Update Recommendations:** 현재 시장 데이터를 분석하여 추천 코인 목록을 수동으로 갱신합니다.
*   **🎓 Retrain AI (New):** 축적된 매매 데이터를 바탕으로 AI 모델을 즉시 재학습시킵니다.
    *   *데이터가 30건 이상 쌓였을 때 효과적입니다.*
    *   *버튼을 누르면 약 1~5초 후 학습 완료 알림이 뜹니다.*

---

## 2. 매매 설정 (Trading Settings)

매매 파라미터를 실시간으로 조정할 수 있습니다. 
**주의:** 조정 후 반드시 **`Commit & Apply`** 버튼을 눌러야 봇에 적용됩니다.

### 💰 Trade Amount (1회 매수 금액)
*   한 종목을 매수할 때 사용할 기본 금액입니다.
*   **유동적 금액:** 데이터가 충분히 쌓이면(30회 이상 매매), 봇이 승률에 따라 이 금액을 기준으로 **자동으로 비중을 조절(Kelly Criterion)**합니다.
    *   *승률 좋을 때:* 설정액보다 더 많이 매수
    *   *승률 나쁠 때:* 설정액보다 적게 매수
*   **최소 금액 제한:** 업비트 정책상 최소 **5,000 KRW** 이상이어야 합니다. 5,000원 미만으로 설정해도 자동으로 5,002원으로 보정되어 주문이 나갑니다.

### 🎯 Target Profit (목표 수익률)
*   이 수익률에 도달하면 매도 체크를 시작합니다.
*   단, **Trailing Stop** 기능이 켜져 있으므로, 가격이 계속 오르면 바로 팔지 않고 따라가다가 꺾일 때 팝니다.
*   *추천값: 0.8% ~ 2.0% (스캘핑 시)*

### 🛑 Stop Loss (손절률)
*   이 비율만큼 손실이 발생하면 즉시 매도하여 손실을 확정합니다.

### 🔄 Rebuy Threshold (재매수 하락폭)
*   익절 후 다시 매수할 때, 판 가격보다 얼마나 더 떨어져야 살지 결정합니다.
*   예: 1,000원에 팔고 Rebuy=1.5%면, 985원 밑으로 내려오면 재매수 고려.

---

## 3. 프리셋 (Quick Presets)

자주 쓰는 설정을 한 번에 적용합니다.

*   **⚡ Scalping:** 짧게 먹고 빠지는 전략 (Target 0.8% / Stop 1.5%)
*   **🛡️ Swing:** 며칠 들고 가는 전략 (Target 3.0% / Stop 5.0%)
*   **🚀 Bull Market:** 불장에서 길게 먹는 전략 (Target 10% / Stop 10%)

---

## 4. 자주 묻는 질문 (FAQ)

### Q. 설정을 바꿨는데 5,000원만 매수돼요.
A. 데이터가 아직 충분하지 않거나(30회 미만), 계좌 잔고가 부족해서 설정값대로 못 사는 경우일 수 있습니다. 또한 업비트 최소 주문 금액(5,000원) 제한 때문일 수도 있습니다. 이제 패치되어 **데이터가 부족해도 설정한 금액(예: 10,000원)대로 매수**하도록 수정되었습니다. (단, 5,000원 이상이어야 함)

### Q. Retrain 버튼이 안 보여요.
A. 최신 버전으로 업데이트(`git pull`)하고 프론트엔드를 재시작(`npm run dev`)해 주세요. 컨트롤 패널에 노란색 버튼으로 추가되었습니다.
